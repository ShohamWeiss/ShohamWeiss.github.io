R6

In this notebook I implemented 2 versions of environments to use the genetic algorithm on, one with rule 2 and one without rule 2. In both environments, the genetic algorithm pretty quickly was able to find an optimal makespan for the job-tasks its was given. However, when compared with a random sequence the fitness of the best makespans did not perform very much significantly better. This is because there is not a huge variance in the resulting fitness from the variance in the sequence pattern. If we add more rules that create a bigger effect on the final makespan length, than the genetic algorithm would show a bigger gap from the random sequence. For example, if each machine performed at a different rate. Also we see that as the number of tasks increase, the more important the genetic algorithm becomes. Either way the genetic algorith is a quick and not very costly algorithm which makes it very powerfull.
