Introduction:

The goal is to have a 3D printed mount on the bolt lock on our door which we could control from our phones. Thus, giving us access to lock and unlock the door with a click of a button. We also want to be able to see whether the door is locked or unlocked through our phones as well as have the data of when the door is locked or unlocked saved for later data analysis. Lastly, we hope to have enhanced security assurance by designing additional features such as electrical access control and camera surveillance system.

Software Design:

General Design:

The client side consists of the HTML page styled by the css and controlled by the javascript. The javascript controls what the client sees on the HTML and communicates with the PHP on the server. On the server side, the PHP was initially meant to run the GPIOs on the raspberry pi, thus moving the servo. And talk with the mariaDB database, but raspbian permissions were blocking the PHP running on the apache2 server. We spent a lot of time trying to fix this, changing permissions and modes moving file placements, etc, but nothing worked. Instead, we connect the PHP to python scripts that easily control the GPIOs and can talk to the database. The communication is two-way for all connections. Response from python is with “print” and from  PHP is with “echo”. The idea of this setup is to have a safe door lock. Where nowhere in the system is the username or password hard coded, and there is no access to the lock and unlock buttons unless the database confirms the username and password. The database (mariaDB) is on the server side and is protected.

When trying to log in, the javascript sends the username and password as a post to the PHP on the server, the PHP calls a python code passing the username and password. This python code makes a connection with the database, reads and checks if the username and password are in there. It then responds to the PHP with “print”. The PHP forwards the message to the javascript with “echo”. The javascript receives the response and acts accordingly. If the username and password are correct, the interface changes to the lock/ unlock interface. If it is wrong the user gets an indication that the username and password are wrong.

When locking / unlocking the door, a similar process occurs, except the python code writes to the database. It writes the new status of the door lock, and then runs the servo to move to the desired position (lock/ unlock). This python code does not respond and the process ends.

Database:

The users table contains our usernames and passwords. This is really just for the first time we use the site. Once we logged in cookies keep us logged in. If we feel we need an upgrade in security, we can change the timeout on these cookies. The status table is used in two ways. First, it records the changes made to the lock. This can be later used for data analysis. Second, when we log in, we want to see the latest status of the lock, this database provides this ability. We check the latest entry and we can display the correct option (lock or unlock).

Step by Step Guide:

Setting up Raspberry pi:

  1. Update rpi:
  1. sudo apt update
  2. sudo apt upgrade (This can take a while, suggested to have ethernet connection)
  3. sudo apt update
  1. Changing keyboard layout to US (from UK):
  1. sudo raspi-config
  2. Choose Localisation Options.
  3. Choose Change Keyboard Layout.
  4. Choose your keyboard or one of the generics.
  5. Choose your language. For instance, for American English choose English (US)
  1. Allowing SSH and Camera:
  1. sudo raspi-config
  2. Choose Interface options
  3. Choose enable SSH
  4. sudo  systemctl start ssh
  5. ssh pi@***IPaddress***
  6. Choose enable Camera and reboot
  1. Setting up wifi (for rpi 3B+ wifi module is built in):
  1. Choose wifi country (US)
  2. Connect to designated wifi
  1. Setting up git:
  1. RPi comes with git installed, we just need to configure
  2. git config --global user.name "Harry Potter"

git config --global user.email "h.potter@hogwarts.prof"

  1. git clone https://github.com/youlink.git
  2. git init
  3. git remote add origin git@github.com:HarryPotter/snitch-sniffer.git
  4. cd ..
  5. chown pi snitch-sniffer → chown [USER] [FILENAME] (to change permissions)

Setting up a server on the raspberry pi:

  1. Installing and setting up apache2:
  1. sudo apt install apache2
  2. sudo chown -R pi:www-data /var/www/html/
  3. sudo chmod -R 770 /var/www/html/
  1. Check if apache2 is working:
  1. Open the Raspberry web browser
  2. Go to “http://127.0.0.1”
  3. You should get a page with a message like “It works! “And plenty of other text
  1. Downloading PHP:
  1. sudo apt install php php-mbstring
  1. Check if PHP is working (changes html apache2 page with PHP page):
  1. sudo rm /var/www/html/index.html
  2. echo "<?php phpinfo ();?>" > /var/www/html/index.php
  3. Go to “http://127.0.0.1”
  4. You should get a page similar to the following:
  1. Downloading mySQL (mariadb):
  1. sudo apt install mariadb-server php-mysql
  1. Check if mySQL is working:
  1. sudo mysql --user=root
  2. You should see that you accessed mySQL
  1. Fixing mySQL to run through the PHP:
  1. sudo mysql --user=root
  2. Inside mySQL:
  1. DROP USER 'root'@'localhost';
  2. CREATE USER 'root'@'localhost' IDENTIFIED BY 'password';
  3. GRANT ALL PRIVILEGES ON *.* TO 'root'@'localhost' WITH GRANT OPTION;
  1. Getting Python mysql connector library:
  1. sudo apt-get -y install python-mysql.connector
  1. Getting Python GPIO library:
  1. sudo apt-get install RPi.gpio

NOTE: Apache2 does not have permissions to run python code which controls GPIOs. There is a way to give permissions to the filename.py file, and there is a way to give apache2 all permissions. But we found it easier to simply move the filename.py to the desktop of the raspberry pi, and save it there. Then when PHP calls it, write the full path “/home/desktop/filename.py”. This way it is not “running” on apache2 and seems to work.

The rest is to code the desired product.

  1. Set up the database with the desired tables.
  2. Set up the html as desired with css, javascript, and jQuery attached.
  3. Set up the javascript to call PHP files, passing the necessary variables.
  4. Have PHP files calling python code and responding to the javascript.
  5. Include gpio and dbconnection libraries in python.
  6. Have the python code connect to the database and run queries as well as run gpio connections to run the servo. Have the python code respond to the PHP.

NOTE: With each step, print everything on your server to make sure that the right connections are made in order. Add the python only when the PHP communicates with the javascript well. Add the database connection and gpio only when the python communicates with the javascript well.

Additional  Features

Camera:

We further improve the security level of our doorlock system by adding a camera that can take snapshots of the visitors (authorized/unauthorized). Specifically, we will take photos whenever a visitor tries to enter the apartment with valid/invalid key fob/card. Once someone tries to enter the building with a key fob, our program will activate the Pi camera and take a snapshot of the person. The photo will be (i) stored locally in the visitor database with timestamp recorded and (ii) uploaded to doorlock host such that users will be able to know the facial information of the visitors at their apartment.

Step by Step:

Step 0: Enable camera interface on RPi by sudo raspi-config

Step 1: Run pip install picamera to install picamera packages

Step 2: Run raspistill -o cam.jpg to test whether the camera is functioning properly. The image will be saved as cam.jpg

Step 3: Write python script screenshot.py in our GitHub repo. You can specify the directory you want the images to be saved as well as the image size you want in that script. Also, you can specify whether you want the photos to be flipped/rotated. Upon finishing the setup, you will be able to run the screenshot.py to see whether the desired photos are taken.

Step 4: Integrate screenshot.py into the RFID modules such that the photos will be taken automatically once a visitor triggers the RFID reader.

RFID:

An RFID reader is used to allow the users to have additional options to unlock the door. Since an RFID key could possibly be stolen or lost, we also use a camera to take a picture of the user of RFID.

Step by Step:

  1. Enable SPI communication on Raspberry Pi.
  1. First, we need to config Raspberry Pi to enable SPI communication for our RC 522 RFID reader by running sudo raspi-config
  2. To enable SPI communication, we choose 5.Interface option -> P4 SPI -> Yes.
  3. Reboot our Raspberry Pi, and run lsmod | grep spi to check weather SPI is correctly enabled. Spi_bcm2835 should be displayed after running this command.
  1. Wire the RC 522 RFID reader to Raspberry Pi.
  1. We connect SDA pin of RC 522 RFID reader to pin 24 on Raspberry Pi.
  2. We connect SCK pin of RC 522 RFID reader to SCLK pin on Raspberry Pi.
  3. We connect MISO pin of RC 522 RFID reader to MISO pin on Raspberry Pi.
  4. We connect MOSI pin of RC 522 RFID reader to MOSI pin on Raspberry Pi.
  5. We connect 3.3 V pin of RC 522 RFID reader to 3.3 V pin on Raspberry Pi.
  6. We connect ground pin of RC 522 RFID reader to ground pin on Raspberry Pi.
  7. We connect RST pin of RC 522 RFID reader to pin 22 on Raspberry Pi.
  8. We connect the alarm to pin 36 on Raspberry Pi.
  1. Download the required library for RC 522 RFID reader
  1. First, run pip install mfrc522 to install the library needed locally.
  2. Then use PHP script to run pip install mfrc522 again. Otherwise, our webpage will not be able to recognize the library when running our python script using PHP.
  1. Write the python script to control the RFID reader and alarm.
  1. First, config the GPIO pins such that Raspberry Pi could control the pins related to RFID reader.
  2. Import the installed library by running from mfrc522 import SimpleMFRC522. Then an object of the RFID reader could be created by running reader = SimpleMFRC522().
  3. Then the id of the tag/card could be read using rfid, text = reader.read(). After reading the rfid of the tag and card, we set the correct ID number to be ID of tag.
  4. Then each time the RFID reader reads a correct ID, it will unlock the door for five seconds. Then it will lock the door again.
  5. If an invalid ID is presented, the alarm will be turned on.
  6. Each time an ID is presented, the camera will take a picture of the user and store it locally.
