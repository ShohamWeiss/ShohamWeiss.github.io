Introduction

The challenge of matching companies and workers has been long-unsolved, with inefficient roundabout solutions provided by several vendors. Candidates either publish their resumes and hope they will be contacted, or they are tasked with searching through many job boards to try and find jobs they are qualified for and match their aspirations. These processes are currently fully dependent on keyword matching. Both candidates and companies fill out a set of keywords and look for a high enough match. With advancements in language modeling, companies have been better able to filter candidates with intelligent models, on the other hand candidates still have no tools to filter through job board efficiently using language models. This project focuses on solving this problem by creating a language model that can understand a job description and extract 4 attributed about the job: the job title, level of experience required, level of education required, and job type (full-time, part-time, etc.). We treat each attribute as a separate classification task and we use both machine learning and deep learning approaches to tackle these tasks and compare their results.

Background

An online job finder would be very useful for fresh graduates searching for jobs. Even with today’s job finders it can be very tough to find jobs that match graduate’s aspirations and match their skills and qualifications. In a job search if the applicant is searching for a job they will choose the filter as per their requirements but the available filters don’t always do a good job finding appropriate jobs as they are reliant on word tagging. In previous work, job finders were made using database systems where all the data from the applicant is stored in the database system so whenever a filter was used a lookup in the database system was performed, this technique is efficient but not accurate. Another approach to also store resumes in a database for lookup was done to help companies find applicants efficiently without searching for so many people and companies. [6]

Other machine-learning approaches to this problem have been suggested, but they are mostly focused on single categories and do not compare the performance of multiple models. In a previous work ”Deep Learning To Extract Knowledge from Job Descriptions”, the work explores deep Learning techniques to extract the knowledge from the descriptions but they focus on one deep learning technique. [5] In our project we are using  multiple machine learning approaches. The motivation behind other project such as the one mentioned is in to help  the field of HR and Management. However, in our work we are using deep learning to compare different language modeling approaches.

Another project used multi label classifiction and different pre-trained language models which predict job titles from job description, they used a multilingual pre-trained BERT model. [7] All the approaches mentioned above focus on only predicting one feature, while our project focuses on predicting 4 features from a job description.

Models

In this project we compare the results of training several machine learning algorithms and deep learning algorithms on several classification tasks each of a different job attributes based on a job description. For machine learning, we look at multinomial naive bayes, gaussian naive bayes, support vector machine, random forest classifier. For deep learning, we look at a custom transformer model and fine tuning a distilbert model.

Deep Learning

Deep learning has been used for language modeling since the success of the recurrent neural network and more recently the success of the transformer model with the attention mechanism. With this in mind, we train a custom transformer, and fine tune a pre trained transformer and compare their results on the different classification categories. We use pytorch as our chosen deep learning library.

Custom Transformer

Our custom transformer was inspired by the Attention is all you need paper, but is meant to be a smaller more minimal approach. The model consists of word embedding and positional embedding layers, transformer layers, and linear layers as seen in Fig. 2. We initially use an embedding size of 100, 2 transformer layers with, a transformer hidden size of 100, 2 heads, and a dropout of 0.1. We have a fully connected layer from the last transformer flattening out the embeddings per token, and another fully connected layer outputting to one of the n classes. After obtaining some initial results, we try hyper parameter tuning the custom model with several other hyper parameter configurations such as more heads, more transformer layers, bigger embedding, and deeper hidden sizes.


Figure 2: Custom transformer model summary

Transfer Learning

We use a distilbert model as it is a lighter weight version of the original bert model. The bert model is a language model that was trained on several language tasks and has a good understanding of language. The distilbert has outputs that were trained to match the outputs of the bert model but with less parameters. The distilbert model has an embedding layer which takes each token and embeds it into a size 768 vector. This layer also adds in the positional embedding of each token as well as a masks layer. The distilbert then has 6 layers of multihead attention modules. Each multihead attention has 12 heads with a fully connected layer after it. There is also a dropout of 0.2. All of this can be seen in figure 3.

Dataset

Data Scraping

The data needed for extracting job attributes is: a job description and the attributes of the job description as labels. This type of data was not available on Kaggle or other open data sources. Thus, we took a web scraping approach. Indeed.com, a job search website, has filters aligning with the attributes we want to predict as can be seen in figure 4.

Figure 4: Indeed.com filter options

To extract the data from Indeed.com, we wrote a python script that, using the Selenium library, automates searching on Indeed and extracts the job descriptions. The script iterates through the different filters and extracts all the job descriptions for each filter combination. The data extraction ran for 2 days and resulted in the extraction 14814 unique job descriptions with their appropriate job attributes (filters) as labels. It turned out that our data is imbalanced. For each attribute we scraped from Indeed, we have more samples from one class than the others sometimes by a significant amount.

Dataset Description

Our data consists of a job description which is a variable length text field describing a job in different levels of details which acts as the features of our models. Along with the labels of the job description which in our case is the job title, experience level, education level, and job type. Job title is free text fields and we focused on specific jobs and locations as can be seen in the analysis. Experience level is a categorical field with either entry, mid, or senior level. Education level is a categorical field with either bachelors, masters, or doctorate levels. Job type is a categorical field with either full time, part time, contract, internship or temporary levels.

Custom Transformer

Data Preparation

SInce neural networks work with vectors and not text data, we first convert our data to neural network friendly torch tensors. We tokenize our text with a basic english unigram tokenizer and build our vocab of our data which ends up having a size of 65963. We also convert our text categorical label (“entry”,”mid”,”senior”) into a neural network friendly numerical category (0,1,2). We have variable length job descriptions with some having less than 100 tokens to ones with 5000 tokens. Referring to the histogram of our token count distribution as seen in Fig. 6, we see a good place to cut the token length is at 2000 tokens, padding any text with less than 2000 tokens. We have our first layer of our deep learning models perform learned embeddings which convert each token to an embedding vector. We additionally add positional encoding to the semantic embedding as was done in the Attention is All You Need paper. Our data preparation pipeline is seen in Fig. 7.

Training

For training the custom transformer model we use google colab pro as that gives us access to accelerated hardware, specifically the NVIDIA A100 with 40GB of ram. With this hardware we can train 2000 tokens encoded with size 100 vectors split into batches of 64. Since our data is imbalanced, we created a sampler that the data loader uses to ensure each label is represented an equal amount of times, also known as stratified sampling. This approach showed an improvement in prediction distribution for our models and overall weighted f1 score. We train the model for 50 epochs or until the early stopping criteria is met. We had to try several learning rates to get a good improvements and landed on a learning rate of 0.00005 for the larger model. Larger learning rates converged faster with worse performance and lower learning rates stagnated early. We ran hyper parameter tuning to see where the best results lie and saw that we needed to increase the size of the model to get better results. We show the loss and validation loss vs. epochs in figure 16.

Transfer Learning

Since the success of large models, transfer learning has grown in popularity. Large models enable researchers to train on multiple tasks, or more complex and generalized tasks, and then someone else can finetune the trained model on their own data. This has shown to provide comparable or better results than training a model from scratch with significantly less training time. For our classification task, we use the distilbert model which is a distilled version of the original bert model. We use this model as it was trained on a large set of text data and is smaller than the original bert to conserve compute power.

Data Preparation

The distilbert model provided by huggingface also comes with a tokenizer. We therefore, use the preconfigured tokenizer to tokenize our whole dataset. As the bert model was trained on multiple tasks, the output of the tokenizer is more complex with token ids and masks which makes it a more expensive tokenization process. Since it turned out that our scraped data is imbalanced, for each classification task, we use a sampler which samples from each class in a fashion that ensures each batch has an equal distribution of classes. This prevents the model from just classifying the most common class and allows it to learn relevant features from the text.

Training

We load in the pretrained model with an added layer for the classification step. We then iterate through the batches and train the model. Training for 3 epoch was enough to reach the results we were reaching with the custom model, but it took the same amount of time because the distilbert model is quite large and slow relative to the custom model.

Conclusion

In this project, we compare different machine learning approaches on the task of classifying job descriptions with 4 different attributes (Experience Level, Education Level, Job Type, and Job Title). For each task a different machine learning approach provides the best performance.

For predicting experience level, the Gaussian Naive Bayes has the best performance followed by the fine-tuned DistilBert. This results indicates that predicting experience level is a statistical task that can be solved by word count rather than deep language understanding.

For predicting education level, the Random Forrest Classifier has the best performance followed by the support vector machine and again by the finetuned distilbert. This result suggests that deciding on which education level a job description matches is more of an “if else” type of task. This makes sense as some aspects of job descriptions might “upgrade” it to require more education and once it is upgraded it shouldn’t be downgraded again.

For predicting the job type, the Gaussian Naive Bayes and the Finetuned DistilBert both have the best performance. In this case the job type is actually provided in the job description and the model has to simply extract it. It is not surprising that the Gaussian Naive Bayes model gave the job type section of the job description high weights and performed well. And the same goes for the finetuned DistilBert.

For predicting the job title, the Custom Transformer has the best performance followed by the Gaussian Naive Bayes and the finetuned DistilBert. In this case, there are 10 classes with some niche differences between each class. Thus, the task is more challenging which could indicate why a custom transformer would perform the best here.

Overall, we see how simply attempting to classify different attributes from the same job description causes different classification approaches to perform differently. For tasks that can be solved in a more statistical approach the Gaussian Naive Bayes approach seems to be the best and cheapest choice. As the task gets more complicated and more decision based, the Random Forrest Classifier shows its strength. Finally, as the task gets even more complicated with more classes and more subtle differences between classes, the deep learning approaches show their (expensive) strength.
